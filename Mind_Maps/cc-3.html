• Service Models

    • Service Models Overview
      ►
      • Infrastructure as a Service (IaaS)
      • Platform as a Service (PaaS)
      • Software as a Service (SaaS)
    • Infrastructure as a Service (IaaS)
      ►
      • Provision processing, storage, network
      • User controls OS, storage, apps
      • Enabling Techniques
        ►
        • Virtualization
        • Hypervisor for OS abstraction
        • Multi-tenant hardware sharing
      • Resource Management Interface
        ►
        • VMs: Create, suspend, terminate
        • Storage: Allocate, write, read
        • Network: IP allocation, bandwidth
      • System Monitoring Interface
        ►
        • Monitor VM CPU, memory, IO
        • Monitor storage usage
        • Monitor network connectivity
    • Platform as a Service (PaaS)
      ►
      • Deploy apps using provider's tools
      • No control over infrastructure
      • Enabling Techniques
        ►
        • Runtime environment design
        • Program libraries & APIs
        • Integrated development tools
      • Programming IDE
        ►
        • Includes debugger, test env
        • Common APIs: compute, storage, communication
      • System Control Interface
        ►
        • Policy-based decision making
        • Workflow control for setup
    • Software as a Service (SaaS)
      ►
      • Access provider apps via browser
      • No infrastructure control
      • Enabling Techniques
        ►
        • Web services
        • Internet-based delivery
        • Cross-device support
      • Web-based Applications
        ►
        • General: Office, messaging
        • Business: ERP, CRM
        • Scientific: Simulations
        • Government: Healthcare, transport
      • Web Portals
        ►
        • Search, news, stock, mail
        • Unified look & access control
