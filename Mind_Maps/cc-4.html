• Deployment Models
    ►
    • Deployment Models
      ►
      • Public Cloud
        ►
        • Openly accessible infrastructure
        • Multi-tenant and external
        • Benefits
          ►
          • Cost Effective
          • Reliability
          • Flexibility
          • Location Independent
          • Utility Style (Pay per use)
          • High Scalability
        • Disadvantages
          ►
          • Low Security
          • Less Customizable
      • Private Cloud
        ►
        • Dedicated to one organization
        • Managed internally or by third party
        • Benefits
          ►
          • High Security and Privacy
          • More Control
        • Disadvantages
          ►
          • Restricted Scope
          • High Cost
          • Limited Scalability
      • Public vs Private
        ►
        • Infra: Homogeneous vs Heterogeneous
        • Policy: Common vs Customized
        • Resources: Shared vs Dedicated
        • Cost: OPEX vs CAPEX
        • Economy: Scale vs Control
      • Community Cloud
        ►
        • Shared among organizations with similar goals
        • Managed internally or by third party
        • Benefits
          ►
          • Cost Effective
          • Organizational Sharing
          • Moderate Security
      • Hybrid Cloud
        ►
        • Combines 2+ clouds (public, private, community)
        • Critical vs Non-critical task separation
        • Benefits
          ►
          • Scalability
          • Flexibility
          • Cost Effective
          • Enhanced Security
        • Disadvantages
          ►
          • Network Complexity
          • Security Compliance Issues
          • Dependent Infrastructure
      • Tenancy
        ►
        • Tenant: Group with shared privileges
        • Resources isolated for each tenant
      • Multitenancy
        ►
        • Multiple users share same infra
        • Data and sessions are isolated
        • Used in both public and private cloud
